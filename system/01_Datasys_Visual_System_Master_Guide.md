# 🎨 데이터시스 비주얼 시스템 마스터 가이드 (v1.0)

## 🚀 1. 목표 및 적용 범위
본 가이드는 데이터시스의 모든 마케팅 및 컨설팅 콘텐츠(웹사이트, 인스타그램, 유튜브, 백서 등)에 걸쳐 시각적 일관성(Visual Consistency)을 확보하는 것을 목표로 합니다. 모든 에이전트 및 디자이너는 이 가이드라인을 최우선 기준으로 삼아 작업을 진행해야 합니다.

## 🎨 2. 브랜드 아이덴티티 (Brand Identity)
### 2.1. 컬러 팔레트 (Color Palette)
| 용도 | 이름 | HEX 코드 | RGB | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **Primary** | 데이터시스 메인 | #004D99 | (0, 77, 153) | 신뢰, 전문성 (브랜드 주조색) |
| **Secondary** | 강조/액센트 | #FF6B6B | (255, 107, 107) | 경고, CTA, 중요 지점 (에러 컬러 활용 가능) |
| **Background** | 기본 배경 | #F7F9FA | (247, 249, 250) | 깨끗하고 가벼운 느낌 |
| **Text/Dark** | 본문 텍스트 | #333333 | (51, 51, 51) | 가독성이 높은 어두운 회색 |
| **Success** | 성공/완료 | #4CAF50 | (76, 175, 80) | 긍정적 피드백 |

### 2.2. 타이포그래피 (Typography)
*   **주요 서체 (Headline/Title):** Pretendard (굵고 명확한 느낌의 산세리프)
    *   사용 규칙: H1 (48pt Bold), H2 (32pt SemiBold), Body (16pt Regular)
*   **보조 서체 (Body/Code):** Noto Sans KR (가독성 높은 고딕 계열)
    *   사용 규칙: 코드 블록 및 장문 설명에 활용.

## 🖥️ 3. 컴포넌트 시스템 (Component System)

### 3.1. CTA 버튼 (Call-to-Action Button)
*   **표준 형태:** 사각형 모서리 라운딩 (Border-radius: 8px)
*   **기본 스타일:** Primary Color Fill (#004D99), Text Color: White.
*   **호버/클릭 상태:** Secondary Color Fill (#FF6B6B), Text Color: White.
*   **적용 좌표 (Web):** [X: 100px, Y: 400px, W: 250px, H: 50px] (예시)

### 3.2. 썸네일/포스트 레이아웃 (Thumbnail/Post Layout)
*   **규격 (Instagram):** 1080px x 1080px (정사각형)
*   **핵심 구조:**
    1.  **헤더 영역 (Header):** 상단 15% - [제목/핵심 키워드] 배치. (배경색: Primary Color의 20% 투명도)
    2.  **메인 비주얼 (Visual):** 중앙 70% - 데이터/이미지 배치. (최대 가독성 확보)
    3.  **출처/로고 (Footer):** 하단 15% - [데이터시스 로고] 및 간결한 출처 표기. (텍스트 크기 12pt)

### 3.3. 인포그래픽/데이터 시각화 (Data Visualization)
*   **차트:** 3D 효과 금지. 플랫 디자인(Flat Design) 필수.
*   **색상 매핑:** 데이터 종류별로 미리 정의된 색상 사용 (예: A그룹은 Primary, B그룹은 Secondary).
*   **표:** 반드시 흑백 명암 대비를 활용하여 강조할 부분에만 Accent Color를 사용합니다.

## 💾 4. 가이드라인 적용 원칙
*   **최소주의(Minimalism):** 모든 디자인은 정보 전달에 초점을 맞추며, 불필요한 장식 요소를 제거합니다.
*   **계층 구조(Hierarchy):** 크기, 색상, 여백을 통해 정보의 중요도를 명확히 구분합니다. (가장 중요한 정보는 가장 눈에 띄어야 합니다.)
*   **반응성(Responsiveness):** 모든 레이아웃은 모바일/데스크톱 등 다양한 환경에서 깨지지 않고 확장 가능해야 합니다.
