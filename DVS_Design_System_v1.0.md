# 📊 데이터 시각화 시스템 (DVS) 디자인 시스템 가이드라인 v1.0

## 📄 목표 및 적용 범위
*   **목표:** 모든 산업별 비교 데이터 분석 시, 일관되고 전문적이며 강력하게 임팩트를 전달하는 표준 시각화 시스템 구축.
*   **적용 범위:** 백서, 웹사이트 데이터 분석 페이지, 프레젠테이션(PPT), 컨설팅 보고서.

## 🎨 1. 컬러 팔레트 정의 (Color Palette)
데이터 시각화의 핵심은 '비교'와 '개선'을 명확히 하는 것입니다.

| 역할 | 색상 코드 (HEX) | 명칭 | 사용 목적 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **Baseline (현황)** | `#AAAAAA` | Neutral Grey | 개선 전의 현 상태, 비교 대상. | 채도가 낮아 주목도가 낮음. |
| **Goal/Solution (개선)** | `#007BFF` | Primary Blue | 우리가 제공하는 솔루션의 목표치. | 메인 브랜드 컬러 사용. |
| **Impact (개선됨)** | `#28A745` | Success Green | 자동화 적용 후의 긍정적 변화, ROI 달성치. | 가장 주목받아야 할 액센트 컬러. |
| **Warning/Risk** | `#FFC107` | Warning Yellow | 개선이 필요한 위험 요소, 주의 지점. | 텍스트 강조 또는 경고 차트. |
| **텍스트 기본** | `#212529` | Dark Text | 본문 및 축 레이블. | |

## 📐 2. 레이아웃 및 그리드 시스템 (Layout & Grid)
*   **기본 그리드:** 12-Column Grid System (표준 웹/프레젠테이션 레이아웃).
*   **데이터 카드 컴포넌트:** 모든 데이터는 3개 또는 4개의 모듈형 '데이터 카드'에 담겨야 합니다.
    *   **구조:** [KPI Title] $\rightarrow$ [Current Value (Baseline)] $\rightarrow$ [Improved Value (Solution)] $\rightarrow$ [Impact % (Gap)].
    *   **강조:** 개선된 값(Impact)이 항상 가장 크고, 액센트 컬러로 강조되어야 합니다.

## 📈 3. 표준 차트 컴포넌트 (Standard Charts)
데이터의 유형에 따라 아래 3가지 표준 차트만 사용하도록 제한합니다.

1.  **비교형 바 차트 (Comparative Bar Chart):**
    *   **용도:** Baseline과 Solution 간의 절대적 수치 차이 비교 (예: A 산업 vs B 산업의 평균 오류율).
    *   **디자인 규칙:** Baseline 막대는 회색, Solution 막대는 `Success Green`으로 명확히 구분합니다.
2.  **트렌드 라인 차트 (Trend Line Chart):**
    *   **용도:** 시간의 흐름에 따른 변화 추이 분석 (예: 3년 간의 비용 변화).
    *   **디자인 규칙:** 트렌드 곡선이 우상향하는 추세일 경우, 솔루션 적용 지점부터의 선은 `Success Green`으로 강조합니다.
3.  **게이지/도넛 차트 (Gauge/Donut Chart):**
    *   **용도:** 특정 KPI 대비 달성률 표시 (예: 목표 대비 95% 달성).
    *   **디자인 규칙:** 중앙에 큰 숫자로 달성률을 표시하고, 나머지 도넛 부분에 `Impact %`를 시각화합니다.

## ✨ 4. 사용자 경험 (UX) 가이드
*   **데이터 간의 연결성:** 차트 제목이나 설명 텍스트에는 반드시 **"데이터 시각화가 전달하는 비즈니스 메시지(The Takeaway)"**를 굵은 글씨로 요약하여 명시해야 합니다.
*   **정보 밀도:** 한 페이지에 너무 많은 차트를 넣지 않도록 주의하고, 각 차트 세트당 하나의 핵심 메시지만 전달하도록 레이아웃을 분리합니다.
